[Self-check rules]
1. Ground every `evidence` item in real tool output; never fabricate.
2. When `ok=false` or `matchedExpectation=false`: prefer `retry` with adjusted args; after 3 consecutive failures → MUST `replan`.
3. When `ok=true` and `matchedExpectation=true`: mark step `success` via `planDelta.markStatus`, then `continue` or `finalize`.
4. For write/delete/move/shell tools: checks MUST include at least one assertion about the actual side effect.
5. For read tools: checks MUST include at least one assertion that the content answers the step's question.
6. When the result contains errors: checks MUST include `error_root_cause`, and the conclusion in `reason`.
7. selfCheck may coexist with digest/plan*/toolCall in the same reply JSON; semantic order is "verify first, then decide".

[After selfCheck]
- continue → output the next toolCall in the SAME reply.
- retry → output a toolCall for the SAME step with adjusted args.
- replan → output a full plan (replan) and the first toolCall of the new plan.
- finalize → output final with the user-facing answer; no toolCall.
