[NORMALIZATION RULES]
1. Strip narrative framing ("As a senior dev..." / "We always...") — keep ONLY the actionable imperative.
2. Split compound rules ("Use Kotlin coroutines AND write tests" → two rules).
3. Deduplicate by semantic meaning, not just text match.
4. Resolve conflicts by preferring the more specific scope; on equal scope, prefer "must" over "should" over "may".
5. Drop rules that conflict with [BASE_SYSTEM] cardinal rules (e.g., "never preserve style" — DROP and record under `dropped`).
6. Map fuzzy verbs: "prefer" → should, "avoid" → forbid, "always/never" → must/forbid, "consider" → may.
7. Keep each rule.text under 200 characters. If a rule needs more context, move it to `examples`.

[OUTPUT LANGUAGE]
- All user-facing text (rule.text, examples, conflicts.reason, dropped.snippet) in {{userLocale}}.
- scope identifiers, subject taxonomy and kind enums stay in English.
