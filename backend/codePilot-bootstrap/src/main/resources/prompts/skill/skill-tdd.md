name: tdd
description: Follow a test-driven development workflow: write tests first, then implement

## Steps

1. Read the task requirements.
2. WRITE the test(s) first using fs.write or apply_patch.
3. Run the test(s) to confirm they FAIL.
4. IMPLEMENT the feature by modifying code.
5. Run the test(s) to confirm they PASS.
6. Refactor / clean up if needed.
7. Run all tests to verify nothing is broken.
