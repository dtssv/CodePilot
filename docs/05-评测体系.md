# 05 — 评测体系

> 更新日期：2026-05-22
> 本文档基于实际代码实现与对标 Cursor 分析编写。评测口径：**Designed → Implemented → Integrated → Productized**

---

## 1. 评测方法论

### 1.1 四级成熟度

| 级别 | 定义 | 验证方式 |
|---|---|---|
| **Designed** | 设计文档已定义，代码未实现 | 设计评审 |
| **Implemented** | 代码已存在，可编译运行 | 单元测试 / 集成测试 |
| **Integrated** | 已接入主产品链路，与其他模块联动 | 端到端测试 |
| **Productized** | 用户可稳定感知，可作为产品卖点 | 用户验收测试 / 灰度验证 |

### 1.2 评测维度与权重

| 维度 | 权重 | 说明 |
|---|---|---|
| Chat 对话 | 10% | SSE流式+压缩+追问+多模态+搜索+分支+成本感知 |
| Agent 模式 | 15% | Graph编排+断点恢复+自检+修复+用户介入 |
| 代码补全 | 15% | FIM+多候选排序+Tab预测+多行扩展+延迟 |
| 全工程索引+@引用 | 15% | 混合搜索+8种引用+增量索引+结果卡片化 |
| 多文件Diff | 10% | per-hunk审阅+3-way merge+Shadow验证+批量回滚 |
| 工具执行 | 8% | 线程安全+并行+缓存+策略链+命令安全 |
| MCP/Skill | 7% | Skill路由+双域分治+商城+确认门 |
| 安全 | 5% | 脱敏+泄露防护+审计+限流+CSP+Privacy Mode |
| 上下文工程 | 5% | 预算裁剪+压缩+引用解析+token估算 |
| UX | 5% | 组件完整度+交互一致性+主题+国际化 |
| 可观测性 | 5% | 指标+Trace+审计+预算事件 |

**核心结论**：P0 主链「能跑、能看、能审」已达标。主要差距在于感知层（Productized），集成层（Integrated）已基本完成。
