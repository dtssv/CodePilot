## 1. 产品定位

CodePilot 是面向 JetBrains IDEA 的 AI 编码助手，提供 Chat 对话、Agent 自动执行、代码补全、内联编辑、重构/评审/注释/测试/文档等一站式 AI 编码体验。所有模型调用经由自研 Java 后端中转，支持 MCP/Skill 商城、Graph 编排、断点续跑、本地化会话存储与多语言场景。

## 4. SSE 事件双层架构

### 4.1 底层事件（协议层，机器消费）✅

| 事件 | 方向 | 用途 |
|---|---|---|
| `tool_call` | 后端→插件 | 请求客户端执行工具 |
| `tool_result_ack` | 插件→后端 | 工具执行结果回传 |
| `graph_plan` | 后端→插件 | Graph内部执行阶段 |
| `graph_transition` | 后端→插件 | 节点跳转通知 |
| `graph_info_request` | 后端→插件 | 信息收集请求 |
| `graph_info_result` | 插件→后端 | 信息收集结果回传 |
| `user_plan` | 后端→插件 | 用户可见的Plan步骤 |
| `user_plan_progress` | 后端→插件 | Plan步骤状态更新 |

### 4.2 语义层（交互层，用户消费）✅

| 事件 | 方向 | 渲染方式 |
|---|---|---|
| `agent_thinking` | 后端→前端 | AgentStepCard(thinking)：意图声明 |
| `agent_reading` | 后端→前端 | AgentStepCard(reading)：文件读取声明 |
| `agent_writing` | 后端→前端 | AgentStepCard(writing)：文件变更预览 |
| `agent_running` | 后端→前端 | AgentStepCard(running)：命令执行声明 |
| `delta` | 后端→前端 | AgentContentRenderer：Markdown/plan/file/run结构化渲染 |
| `done` | 后端→前端 | 流结束信号，含reason和continuationToken |

### 4.3 v2 结构化事件 ✅

EventEnvelope 三元组定位：`seq + turnId + stepId`，允许乱序、补传、断点续传。

- EventBus：Project服务，生成单调seq，5000条ring buffer，replaySince
- turnReducer：纯函数reducer，处理顺序/乱序/replay路径
- ChatViewV2：v2默认启用的聊天视图

## 5. Graph 引擎拓扑

### 5.1 标准模式 ✅

```
START → intake → planning → preCheck → generate → applyPatch → verify
              │         │          │          │           │          │
              │      finalize   gather     gather      repair     repair
              │                   │          │           │          │
              │                reenter    reenter    applyPatch   askUser
              │
              └→ (resume时跳到目标节点)

commit → preCheck(下一phase) / finalize(全部完成)
askUser → 任意目标节点 / END
finalize → END
```

### 5.2 Deep Research 模式 ✅

```
START → intake → planning → generate → gather → reenter → searchEvaluate
                                                      │
                                              sufficient?
                                             /           \
                                           yes            no
                                           /                \
                                       commit          generate(重试)
                                       │
                               more phases?
                              /           \
                            yes            no
                           /                \
                      generate          synthesize → finalize → END
```

## 6. 关键设计原则

1. **Graph Engine 替代 AgentLoop** ✅：StateGraph 声明式编排，启动时编译一次，每次请求 invoke，支持条件边动态路由
2. **统一对话端点** ✅：Chat与Agent共用 `/v1/conversation/run`，由 `mode` 字段路由
3. **双层Plan架构** ✅：UserPlan（用户可见）+ ExecutionPhases（Graph内部驱动）
4. **SSE事件双层分离** ✅：底层供机器消费，语义层供用户消费
5. **ToolResultBus异步回传** ✅：CompletableFuture + Redis Pub/Sub 双机制
6. **ContextStore占位符机制** ✅：全量代码存本地Map，消息仅传 `\u0001ctxId` 引用
7. **ToolDispatcher线程安全** ✅：读操作走 ReadAction，写操作走 WriteCommandAction
8. **后端不落库用户会话/代码** ✅：仅做转发与编排，checkpoint仅Redis短TTL缓存
9. **Prompt模板版本化** ✅：20个必需段，PromptRegistry启动时加载，永不暴露给客户端
10. **写操作必须经过Diff预览+用户确认** ✅：PatchStaging + per-hunk Accept/Reject
11. **MCP已集成对话** ✅：Graph `mcp.call` + 执行前确认门(McpConfirmGate)
12. **代码索引纯本地** ✅：BM25(40%)+语义(30%)+符号(20%)+路径(10%)混合评分
13. **可中断+可恢复** ✅：AskUser节点抛出GraphInterruptException，保存checkpoint到Redis
14. **App.tsx已瘦身** ✅：~240行顶层布局组件，状态逻辑下沉至独立store

## 7. 通信架构

### 7.1 WebUI ↔ Plugin 双向桥 ✅

```
Web → Plugin: sendToPlugin(type, payload) → cefQuery
Plugin → Web: window.__codepilot_dispatch(type, payload)
```

### 7.2 Plugin ↔ Backend SSE ✅

```
Plugin → Backend: POST /v1/conversation/run (OkHttp + SSE)
Backend → Plugin: SSE事件流 (delta/tool_call/agent_thinking/...)
Plugin → Backend: POST /v1/conversation/tool-result (工具结果回传)
```
