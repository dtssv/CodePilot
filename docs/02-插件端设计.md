## 2. CefChatPanel（主聊天面板）

### 2.1 架构

```
┌─ CefChatPanel (JPanel) ─────────────────────────────┐
│  JBCefBrowser ←── React/Vite WebUI (index.html)     │
│       ↕ CefMessageRouter (双向桥接)                   │
│  Plugin → Web: executeJavaScript(                    │
│    "window.__codepilot_dispatch('eventType', payload)")│
│  Web → Plugin: cefQuery({                            │
│    request: JSON.stringify({type, payload}) })        │
├──────────────────────────────────────────────────────┤
│  ConversationClient (SSE客户端)                       │
│  SessionStore (本地持久化)                             │
│  ToolDispatcher (工具路由)                             │
│  GatherDispatcher (信息收集路由)                       │
│  GraphStateStore (Graph状态缓存)                      │
│  ContextStore (代码占位符Map)                          │
└──────────────────────────────────────────────────────┘
```

### 2.2 SSE事件转发架构

```
后端 SSE → ConversationClient.onEvent()
  → CefChatPanel.handleSseEvent()
    → dispatchToWeb(eventType, payload)
      → browser.executeJavaScript(
            "window.__codepilot_dispatch('eventType', payload)")
        → React App.tsx onPluginEvent()
          → setMessages / setAgentSteps / setPlanSteps 更新UI
```

### 2.3 Web→Plugin事件处理

```
Web UI cefQuery({type, payload})
  → CefChatPanel.handleWebMessage()
    → 路由到对应handler:
      ├── "send_message" → ConversationClient.run()
      ├── "apply_patches" → PatchApplier.applySelectedHunks()
      ├── "reject_patches" → 记录到session
      ├── "tool_result" → ConversationClient.postToolResult()
      ├── "at_suggest" → AtReferenceProvider.suggest()
      ├── "at_resolve" → AtReferenceProvider.resolve()
      └── ...
```

## 5. 插件端工具体系

### 5.1 ToolDispatcher

路由模型tool_call到本地实现，线程安全约束：

| 工具类别 | 执行方式 | 线程要求 |
|---|---|---|
| fs.read / fs.list / fs.search / fs.outline | `runReadAction` | ReadAction |
| fs.write / fs.replace / fs.create / fs.delete / fs.move | `WriteCommandAction` | WriteAction |
| shell.exec | `GeneralCommandLine` + `CapturingProcessHandler` | 后台线程 |
| ide.openFile / ide.diagnostics | `runReadAction` | ReadAction |
| ide.applyPatch | `WriteCommandAction` | WriteAction |
| mcp.\<server\>.\<tool\> | `McpProcessManager` | 后台线程 |

**并行调度**：只读工具可并行执行（dispatchParallel），写工具始终串行。

**结果缓存**：只读工具结果缓存30秒（ConcurrentHashMap + TTL），避免重复文件读取。

### 5.2 GatherDispatcher

专门处理信息收集请求（GatherAction发出的graph_info_request）：
- 只执行只读操作：fs.read / fs.list / fs.search / fs.outline / ide.diagnostics
- 全部走ReadAction
- 结果通过graph_info_result事件回传后端

### 5.3 工具清单

| 名称 | 说明 | 风险 |
|---|---|---|
| `fs.read` | 读取文件（可指定行范围） | low |
| `fs.list` | 列目录 | low |
| `fs.search` | 文本搜索 | low |
| `fs.outline` | PSI outline | low |
| `fs.create` | 新建文件 | medium |
| `fs.write` | 整文件覆盖 | medium |
| `fs.replace` | 局部替换（首选） | medium |
| `fs.delete` | 删除（移到Trash） | high |
| `fs.move` | 移动/重命名 | high |
| `fs.grep` | 正则搜索 | low |
| `shell.exec` | 执行命令 | high |
| `ide.openFile` | 打开文件 | low |
| `ide.diagnostics` | 获取编译错误 | low |
| `ide.applyPatch` | 批量应用Patch | medium/high |

## 6. SessionStore（本地持久化）

### 6.1 目录结构

```
~/.codePilot/sessions/<workspaceHash>/<sessionId>/
├── meta.json          # 标题、模型、模式、创建/更新时间
├── messages.ndjson    # 每行一条消息，append-only
├── events.ndjson      # 完整SSE事件流，append-only（调试用）
├── plan.json          # 最近一次UserPlan快照
├── ledger.json        # 最近一次TaskLedger快照
└── checkpoint.json    # 断点恢复数据
```

### 6.2 双流持久化

| 流 | 文件 | 写入时机 |
|---|---|---|
| 消息流 | messages.ndjson | 每条user/assistant/tool消息 |
| 事件流 | events.ndjson | 每个SSE事件（含tool_call/delta/agent_thinking等） |

事件流用于完整回放和调试，消息流用于上下文组装。

### 6.3 关键API

```kotlin
SessionStore.newSession(workspaceHash, mode, modelId)  // 创建会话
SessionStore.appendMessage(handle, role, content, extra) // 追加消息
SessionStore.appendEvent(handle, event, data)            // 追加事件
SessionStore.savePlan(handle, plan)                      // 保存Plan
SessionStore.loadPlan(handle)                            // 加载Plan
SessionStore.saveCheckpoint(handle, checkpoint)          // 保存断点
```

```kotlin
        selectedHunkIds: Set<String>

        WriteCommandAction.runWriteCommandAction(project) {
            // 同一文件多个hunk按行号从后到前应用
            // 避免偏移量漂移
            // 单hunk内原子，hunk间非原子
        }
    }
}
```

### 7.2 ShadowWorkspace验证（可选）

```
Patch → ShadowWorkspace.validate(patches)
  → 复制受影响文件到临时目录
  → 应用Patch → 运行编译检查
  → 通过 → 允许ApplyPatchAction发tool_call
  → 失败 → 错误回传RepairAction触发修复
```

## 8. SmartMatcher（智能定位）

当Patch的search文本无法精确匹配时，多级匹配策略：

| Phase | 策略 | 复杂度 | 置信度 |
|---|---|---|---|
| 1 | 精确匹配 | O(n) | 1.0 |
| 2 | 空白归一化 | O(n) | 0.85 |
| 3 | 逐行Levenshtein滑动窗口 | O(n*m) | 1.0-normalizedDistance |
| 4 | 行内容hash滑动窗口 | O(n) | 由匹配决定 |

- 置信度≥0.8自动使用，<0.8发needs_input让用户确认
- 总超时500ms

## 11. Bug Finder

- 自动：文件保存debounce 30s
- 手动：右键→CodePilot→Find Bugs
- 流程：DiagnosticCollector收集Inspections → POST /v1/actions/bug-scan → LLM分析 → BugInlayProvider渲染
- 点击Inlay可触发Agent模式修复

## 12. 安全护栏

- 路径必须落在工程根内；越界直接拒绝
- 黑名单：`.git/`、`.idea/`、`node_modules/`、`*.env*`、密钥文件
- `fs.replace`的`expectMatches`不一致→拒绝
- `shell.exec`命令黑名单：`rm -rf /`、`mkfs`、`shutdown`等
- 所有操作走WriteCommandAction + UndoManager，可一键撤销
- 敏感数据脱敏：邮箱/手机号/身份证/AK-SK正则脱敏

## 13. Agent模式时序（Graph Engine驱动）

```
User输入 + mode=agent
  → ConversationClient.run(SSE)
  → 后端GraphEngineService.run()
  → Graph执行：

    intake → planning(LLM调用,skipPlan判断)
      → [emit: agent_thinking, user_plan]
    → preCheck
    → generate(LLM调用)
      → [emit: agent_thinking/agent_writing, delta]
      → 输出patches?
        → applyPatch
          → [emit: tool_call(batch patches)]
          → 插件PatchApplier执行
          → [emit: tool_result_ack]
          → ToolResultBus回传结果
        → verify(可选ShadowWorkspace)
        → commit
          → [emit: graph_phase_done]
      → 输出infoRequests?
        → gather
          → [emit: graph_info_request]
          → GatherDispatcher执行
          → [emit: graph_info_result]
        → reenter → 回到generate
      → 输出askUser?
        → askUser
          → [emit: needs_input]
          → GraphInterruptException → 保存checkpoint
          → 用户回答 → resume

    → finalize → [emit: done]
```
