## 6. 工具体系

### 6.1 ToolDispatcher ✅

| 工具类别 | 执行方式 | 线程要求 |
|---|---|---|
| fs.read / fs.list / fs.search / fs.outline / fs.grep | `runReadAction` | ReadAction |
| fs.write / fs.replace / fs.create / fs.delete / fs.move | `WriteCommandAction` | WriteAction |
| shell.exec | `GeneralCommandLine` + `CapturingProcessHandler` | 后台线程 |
| ide.openFile / ide.diagnostics | `runReadAction` | ReadAction |
| ide.applyPatch | `WriteCommandAction` | WriteAction |
| mcp.\<server\>.\<tool\> | `McpProcessManager` | 后台线程 |

- **并行调度**：只读工具可并行执行(dispatchParallel)，写工具始终串行
- **结果缓存**：只读工具结果缓存30秒（ConcurrentHashMap + TTL）
- **Staging拦截**：`stageBeforeApply=true`时，fs.write/create/replace/delete走暂存流程
- **fs.move**：走RefactoringElementFactory更新引用（MoveRefactoring + RenameRefactoring + PsiSearchHelper import更新）

## 9. SessionStore（本地持久化）✅

### 9.1 目录结构

```
~/.codePilot/sessions/<workspaceHash>/<sessionId>/
├── meta.json          # 标题、模型、模式、创建/更新时间
├── messages.ndjson    # 每行一条消息，append-only
├── events.ndjson      # 完整SSE事件流，append-only（调试用）
├── plan.json          # 最近一次UserPlan快照
├── ledger.json        # 最近一次TaskLedger快照
├── checkpoint.json    # 断点恢复数据
└── branches.json      # 分支信息
```

### 9.2 关键API

```kotlin
SessionStore.newSession(workspaceHash, mode, modelId)
SessionStore.appendMessage(handle, role, content, extra)
SessionStore.appendEvent(handle, event, data)
SessionStore.savePlan(handle, plan)
SessionStore.saveCheckpoint(handle, checkpoint)
SessionStore.forkFromMessage(handle, messageId)   // 分支
SessionStore.listBranches(handle)                  // 列出分支
SessionStore.searchMessages(query, limit)          // 跨会话搜索
```

## 10. PatchApplier + PatchStaging ✅

### 10.1 暂存工作流（stageBeforeApply=true，默认启用）

```
Agent输出patches → ToolDispatcher拦截 → PatchStaging.stage(patches)
  → WebUI ChangePanel展示 → 用户逐hunk Accept/Reject
  → applyFile / applyAll → PatchApplier原子写入
  → undoTurn → PatchRecorder批量回滚
```

### 10.2 SmartMatcher多级匹配

| Phase | 策略 | 置信度 |
|---|---|---|
| 1 | 精确匹配 | 1.0 |
| 2 | 空白归一化 | 0.85 |
| 3 | 逐行Levenshtein滑动窗口 | 1.0-normalizedDistance |
| 4 | 行内容hash滑动窗口 | 由匹配决定 |

置信度≥0.8自动使用，<0.8发needs_input让用户确认。

## 11. 安全护栏 ✅

- 路径必须落在工程根内；越界直接拒绝（PathGuard）
- 黑名单：`.git/`、`.idea/`、`node_modules/`、`*.env*`、密钥文件
- `fs.replace`的`expectMatches`不一致→拒绝
- `shell.exec`命令黑名单：`rm -rf /`、`mkfs`、`shutdown`等
- 所有操作走WriteCommandAction + UndoManager，可一键撤销
- 敏感数据脱敏：邮箱/手机号/身份证/AK-SK正则脱敏（RedactionService）
- HmacSigner请求签名 + nonce去重Redis
