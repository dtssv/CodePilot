## 3. GraphEngine核心 ✅

### 3.1 GraphEngineService

使用 Spring AI Alibaba 的 StateGraph API：

- 启动时编译图，每次请求invoke
- Sinks.Many实时推送SSE事件
- 专用scheduler执行图（避免阻塞ForkJoinPool）
- 标准模式 + Deep Research模式双拓扑
- 支持resume（从checkpoint恢复状态继续执行）

### 3.2 各Action职责与emit事件

## 5. ToolResultBus ✅

### 5.1 双机制架构

```
等待线程(ApplyPatchAction/GatherAction):
  1. registerFuture(sessionId:toolCallId, future)
  2. emit tool_call SSE
  3. future.get(timeout) 等待结果

HTTP Handler(POST /v1/conversation/tool-result):
  1. publish(sessionId, toolCallId, result)
  2. 本实例: complete Future
  3. 跨实例: publish to Redis channel
```

- **CompletableFuture**（主）：ConcurrentHashMap<sessionId:toolCallId, Future>
- **Redis Pub/Sub**（辅）：channel=`codepilot:tool-result:{sessionId}`
- 默认超时120s（ApplyPatch），60s（Gather）

## 8. ContextBudgeter ✅

6阶段递进裁剪策略：

| 阶段 | 策略 | 是否可逆 |
|---|---|---|
| 1 | MUST-KEEP（不裁）：goal/cursor/pinned/completedToolCalls/projectRules | 否 |
| 2 | 丢弃最早recent消息（保留至少2条） | 是 |

## 9. SkillRouter ✅

### 9.1 路由算法

```
1) 候选 = 后端system域候选(triggers匹配) + 请求体userSkill候选
2) 显式白名单过滤 (skills.requested[])
3) 黑名单过滤 (skills.disabled[])
4) 安全约束 (user域禁止override system, 权限校验)
5) 冲突解决 (project>global, system>user)
6) 限额 (lang.primary=1, lang.aux=2, scenario=2)
7) 排序 (source>priority>scope>version)
```

### 9.2 双域分治

- **system域**：驻后端，永不下发到客户端。商城仅返回安全摘要(redactManifestForSystem)
- **user域**：完全位于客户端本地，每次请求上行userSkills[]，后端临时注入用完即丢
